# Wanderleaf Teas - Интернет-магазин чая

Интернет-магазин чая с уникальным функционалом:
- Система "Чайный путешественник" (поиск по странам и регионам)
- Конструктор авторских чайных сборов
- Личный кабинет с историей заказов и рецептами

## Технологический стек

- **Backend**: FastAPI
- **ORM**: TortoiseORM
- **База данных**: SQLite (позже можно перейти на PostgreSQL)
- **Frontend**: HTML, CSS, JavaScript

## Установка и запуск

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Загрузите данные о чаях в базу данных:
```bash
python scripts/load_tea_data.py
```

3. Запустите сервер:
```bash
uvicorn app.main:app --reload
```
или
```bash
python run.py
```

4. Откройте в браузере:
   - Главная страница: http://localhost:8000
   - Каталог чаев: http://localhost:8000/catalog
   - API документация: http://localhost:8000/docs

## Модели данных

### User (Пользователь)
- Основная информация: email, username, пароль
- Персональные данные: имя, фамилия, телефон
- Роль: покупатель, администратор, модератор
- Адрес доставки

### Country & Region (Страны и регионы)
- Страны чайных регионов с описанием традиций
- Регионы/провинции внутри стран
- Географические координаты для карты

### Tea (Чай)
- Связь со страной и регионом
- Тип чая (черный, зеленый и т.д.)
- Вкусы и ароматы (для фильтрации)
- Цены за 20г и 100г
- Рейтинг и отзывы

### Ingredient (Ингредиенты)
- Категории: ягоды, цветы, пряности, фрукты
- Цены и наличие
- Популярность (счетчик использования)

### CustomBlend (Авторский сбор)
- Создан пользователем
- Основа (чай или тип чая)
- Компоненты с пропорциями
- Модерация перед публикацией
- Рейтинг и популярность

### Order (Заказ)
- Статусы: ожидает оплаты, оплачен, отправлен и т.д.
- Элементы заказа (чай или сбор)
- Адрес доставки

### Review (Отзыв)
- На чай или авторский сбор
- Рейтинг 1-5
- Модерация

## Архитектура

Проект использует модульную архитектуру с роутерами FastAPI:

### Структура кода

- **Роутеры** (`app/routers/`) - обработчики для разных частей приложения
  - `teas.py` - каталог чаев, детальные страницы, API для чаев
  - Каждый роутер - отдельный модуль с логикой для определенной области
  - Роутеры подключаются в `app/main.py` через `app.include_router()`
  - В будущем: `countries.py`, `blends.py`, `orders.py` и т.д.

- **Модели** (`app/models/`) - модели базы данных через TortoiseORM
  - Каждая модель в отдельном файле
  - Все модели импортируются в `app/models/__init__.py`
  - Используется SQLite (можно перейти на PostgreSQL)

- **Шаблоны** (`templates/`) - HTML шаблоны с Jinja2
  - `base.html` - базовый шаблон с шапкой и подвалом
  - Все страницы наследуются от `base.html`
  - Используется `app/templates.py` для рендеринга

- **Статика** (`static/`) - CSS, JavaScript, изображения
  - `css/` - стили, разделены по файлам (style.css, header.css, footer.css, catalog.css и т.д.)
  - `js/` - JavaScript файлы для каждой страницы
  - Адаптивный дизайн через CSS медиа-запросы

### Важные детали реализации

1. **Рендеринг шаблонов**: Используется `app/templates.py` с функцией `render_template()`
   - Экспортируется для использования в роутерах
   - Настроен Jinja2 Environment

2. **Работа с фильтрами в формах**: 
   - Query параметры принимаются как `str`, затем преобразуются в `int` или `None`
   - Это решает проблему с пустыми строками из HTML форм

3. **Поиск**: Используется `icontains` для case-insensitive поиска
   - Поиск работает только по названию чая (name и name_en)

4. **Префетчинг данных**: Используется `.prefetch_related()` для загрузки связанных объектов
   - Например: `prefetch_related("country", "region", "tea_type", "flavors")`

5. **Адаптивность**:
   - Desktop (> 968px) - полная версия
   - Tablet (768-968px) - 2 колонки
   - Mobile (< 768px) - 1 колонка, мобильное меню
   - Медиа-запросы в каждом CSS файле

6. **CSS переменные**: Используются CSS custom properties в `style.css`
   - Цвета, отступы, тени определены в `:root`
   - Все файлы используют эти переменные

## Загрузка данных

Скрипт `scripts/load_tea_data.py` загружает данные из `tea_regions_data.json`:
- Создает типы чая
- Создает вкусы и ароматы
- Загружает страны, регионы и чаи
- Связывает чаи с типами и вкусами

Запуск: `python scripts/load_tea_data.py`

**Важно**: Данные нужно загрузить перед первым использованием!

## Стиль кода и конвенции

### Python:
- Используется async/await для всех операций с БД
- Роутеры используют APIRouter из FastAPI
- Query параметры для фильтрации и пагинации
- Контекст передается в шаблоны через словарь

### Frontend:
- Чистый HTML, CSS, JavaScript (без фреймворков)
- CSS переменные для единообразия
- Mobile-first подход (медиа-запросы)
- JavaScript модули для каждой страницы

### База данных:
- SQLite (файл `db.sqlite3`)
- TortoiseORM для работы с БД
- Модели с описаниями полей
- Используются связи (ForeignKey, ManyToMany)
